    PatrowlDashboard API v1

    Scroll down for code samples, example requests and responses. Select a language for code samples from the tabs above or the mobile navigation menu.

    API usage for Patrowl Dashboard

    • API Base URL: https://dashboard.cloud.patrowl.io/api/auth
    • Email: help@patrowl.io
    • License: EULA License

    What can I do with this API ?

    You can use this Patrowl Dashboard API to:

    • List your assets and retrieve the details,
    • List your vulnerabilities and retrieve the details,
    • List your retests and retrieve the details.

    Authentication

    Get your API access token

    Your API access token is accessible on Patrowl Dashboard in your account settings:

    • Go to Patrowl Dashboard > Account > API Token
    • Press the button to generate the API token

    The token can be renewed or deleted from this same page.

    How to use your API access token

    Your API access token has to be sent in every requests, in the Authorization header of the request:

    Authorization: Token <your-API-token>

    GET /assets/

    List all your assets, or filter them according to your needs (by type, criticality, score, etc.).

    Parameters

    Name In Type Required Description
    type query string false Filter by asset type. Ex: ip, ip-range, ip-subnet, fqdn, domain, url or keyword.
    type__icontains query string false Filter by asset type. The parameter is not case sensitive, and performs a containment test.
    criticality query string false Filter by criticality. Ex: 1 for low, 2 for medium or 3 for high.
    description__icontains query string false Filter by description content. The parameter is not case sensitive, and performs a containment test.
    value query string false Filter by value. It can be an IP adresse, an URL, a domain, etc.
    value__icontains query string false Filter by value. The parameter is not case sensitive, and performs a containment test.
    score query number false Filter by exact score (from 0 to 100).
    score__lte query number false Filter asset less than the specified score. Ex: 30 will filter all asset with a score lower or equal 30.
    score__gte query number false Filter asset greater than the specified score. Ex: 30 will filter all asset with a score greater or equal 30.
    search query string false Search in value, type and criticality. The parameter is not case sensitive, and performs a containment test.
    org_id query string false Filter by organization. Your different org_id can be retrieved from /api/auth/orgs.
    is_monitored query string false Filter by monitored asset. true to filter on monitored asset, and false on unmonitored asset.
    tags_value query string false Filter by tags. You can specify multiple tags, separated by a comma ,. Assets returned will have at least one of these tags. Ex: a,b,c.
    tags_value_union query string false Filter by tags. You can specify multiple tags, separated by a comma ,. Assets returned will have all these tags. Ex: a,b,c.
    tags_or query string false Filter by tags. You can specify multiple tags, separated by a comma ,. Assets returned will have at least one of these tags. Ex: a,b,c.
    sorted_by query string false Sort the results. You can sort by: id, value, description, rating, score, type, criticality, created_at or activevulns. Add - in front of the parameter to have descending ordering. Ex: -created_at.
    tags query string false Filter by tags ID.
    page query integer false A page number within the paginated result set.
    limit query integer false Number of results to return per page.
